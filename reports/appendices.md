# Appendices

## Appendix A — Source code and scripts

| Path | Purpose |
|---|---|
| `docker-compose.yml` | Orchestrates MinIO, ClamAV, audit-webhook, and the `mc` admin/test tool on an isolated Docker network |
| `scripts/gen_certs.sh` | Generates the OpenSSL root CA + MinIO server TLS certificate |
| `scripts/setup.sh` | End-to-end bootstrap: certs → containers → RBAC/encryption/versioning → dataset → upload |
| `minio/init/init-minio.sh` | Creates buckets, enables versioning + SSE-S3 encryption, creates IAM policies and the 4 RBAC users |
| `minio/init/upload_dataset.sh` | Mirrors the generated dataset into the matching MinIO buckets |
| `minio/policies/*.json` | IAM policy documents: `hr-readwrite-policy.json`, `finance-readonly-policy.json`, `auditor-readonly-policy.json` |
| `dataset/generate_dataset.py` | Generates the 120,000-record synthetic dataset (HR, finance, contracts) with Faker, seeded for reproducibility |
| `dataset/build_manifest.py` | Computes the SHA-256 integrity manifest (`dataset/output/manifest.sha256.json`) over every shard file |
| `logging/audit-webhook/app.py` | Flask receiver for MinIO audit + server log webhooks; persists JSON Lines and exposes `GET /events` for querying |
| `security-tests/common.sh` | Shared helpers (evidence file creation, `mc` wrapper, PASS/FAIL logging) used by every test script |
| `security-tests/test_01_unauthorized_access.sh` … `test_08_backup_recovery.sh` | The 8 security tests (Part D) |
| `security-tests/run_all_tests.sh` | Runs all 8 tests in sequence and writes a timestamped run summary |
| `security-tests/malware-samples/` | EICAR test file + clean control file used by the malware-scanning test |

## Appendix B — Dataset

- **Generator:** `dataset/generate_dataset.py` (Faker, `--seed 42` for determinism)
- **Volume:** 120,000 records total (40,000 each of HR records, finance invoices, contracts) — exceeds the 100,000-record minimum
- **Format:** newline-delimited JSON (JSONL), sharded into 20 files of 2,000 records per category under `dataset/output/{hr,finance,contracts}-documents/`
- **Integrity baseline:** `dataset/output/manifest.sha256.json` — SHA-256 + byte size for every shard file, generated by `dataset/build_manifest.py` before upload

Sample record (`hr-documents/hr_shard_0001.jsonl`, line 1):

## Appendix C — Configuration files

| File | Contains |
|---|---|
| `.env.example` (copy → `.env`, gitignored) | MinIO root credentials, RBAC test-user credentials, KMS key, port bindings |
| `certs/ca.crt`, `certs/public.crt` | Root CA and server TLS certificate (public; private keys are gitignored) |
| `minio/policies/hr-readwrite-policy.json` | Grants `hr-manager` read/write on `hr-documents` only |
| `minio/policies/finance-readonly-policy.json` | Grants `finance-analyst` read-only on `finance-documents` only |
| `minio/policies/auditor-readonly-policy.json` | Grants `auditor` read-only across all three document buckets |
| `docker-compose.yml` | Service definitions, healthchecks, audit/log webhook wiring, volume mounts |

## Appendix D — Logs and evidence

All raw test output lives in `evidence/` (one timestamped `.log` file per test run, plus JSON audit-event exports, baseline/post-tamper download artifacts, and run summaries). `evidence/screenshots/` additionally holds 27 PNG captures of the MinIO Console (login, buckets, encryption/versioning, IAM policies/users, per-role Object Browser views showing RBAC enforcement) and the audit-webhook `/events` dashboard. See `reports/testing-matrix.md` for the mapping from each test to its evidence file, and [evidence/README.md](../evidence/README.md) for the full evidence-portfolio index (config files, logs, terminal output, screenshots, dashboard/API captures).
